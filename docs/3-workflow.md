### Le concept de traitement des contributions : un process transparent et KISS

Pour que chacun voit comment son idée navigue du formulaire au programme, voilà le flux simple et open (tout tracé sur GitHub pour vérification) :

1. **Arrivée du message** : Via formulaire (anonyme ou nommé), il atterrit dans le github (copié collé) (ou CSV exporté sur GitHub). On vérifie le respect de la charte (pas d'insulte, focalisé bien commun – sinon, on redirige gentiment).

2. **Inclusion dans le programme** : L'idée est taguée par thématique (ex. alimentation : "circuit court"). Si elle s'aligne avec les constats locaux (comme le marché d'Audierne en hiver), elle est ajoutée comme proposition potentielle.

3. **Extraction d'idée originale** : On isole le cœur unique (ex. "Ateliers cuisine bio intergénérationnels" d'un retraité expert en recettes locales) – pas de duplication, on valorise l'originalité.

4. **Recherche appliquée ailleurs** : On vérifie si ça existe (ex. : ateliers similaires à Plouhinec ou Pont-Croix via circuits courts) – pour enrichir sans réinventer la roue. Sources : recherches rapides (comme Ouest-France pour événements locaux).

5. **Synthèse et publication** : Idée intégrée au programme sur le site (avec crédit anonyme si souhaité). Rapport journalier auto (via GitHub Actions) : "Aujourd'hui, 3 idées sur bien-être – voici le top !".

Tout est public, modifiable via Pull Requests – l'IA (comme Capitaine) aide à synthétiser, mais les humains valident. Ça garantit que chaque contribution compte, sans biais partisan.
