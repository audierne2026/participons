# Transparence du projet

Ce projet est conçu pour être **totalement transparent** dès la campagne et au-delà. L’objectif : permettre à chaque habitant d’Audierne-Esquibien de suivre, comprendre et contribuer à l’élaboration du programme municipal pour 2026.

### Nos pratiques de transparence

- **Tout le code et le contenu sont publics**  
  Le site, le programme et les contributions sont hébergés sur GitHub dans le repository public [audierne2026/participons](https://github.com/audierne2026/participons). Rien n’est caché : vous pouvez consulter l’ensemble des fichiers à tout moment.

- **Historique complet et traçable**  
  Chaque modification (ajout au programme, correction, nouvelle idée) est enregistrée dans un **commit** avec un message clair expliquant le pourquoi du changement.  
  Vous pouvez explorer l’historique complet ici :  
  → [Historique des commits](https://github.com/audierne2026/participons/commits/main)

- **Branch dédiée aux travaux en cours**  
  La branche `wip-history` (Work In Progress) contient les versions intermédiaires, les brouillons et les discussions préparatoires. Elle permet de voir comment les idées évoluent avant d’être intégrées dans la version principale.  
  → [Voir la branche wip-history](https://github.com/audierne2026/participons/tree/wip-history)

- **Contributions et décisions expliquées**  
  Chaque changement important fait l’objet :

  - d’un message de commit précis (ex. : « Ajout proposition logement : résidence à l’année suite réunion du 15/01 »),
  - ou d’une Pull Request avec description détaillée du raisonnement.

- **Aucune donnée sensible**  
  Aucune information personnelle ou sensible n’est stockée dans le repository. Les contributions via formulaires sont anonymisées ou publiées avec accord explicite.

### Pourquoi cette transparence ?

Nous souhaitons montrer concrètement que la co-construction promise n’est pas un slogan : **tout se passe sous vos yeux**.  
Si nous sommes élus, ces pratiques seront étendues à la gestion municipale (consultations publiques ouvertes, suivi des décisions, budget participatif via une plateforme comme Decidim).

➡️ Vous avez une question sur une modification ? Ouvrez une **Issue** ou une **Discussion** sur GitHub, ou contactez-nous directement. Chaque voix compte !

Merci de participer à cette démarche ouverte et apaisée.
